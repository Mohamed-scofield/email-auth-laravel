Configuration de Base

1. Fichier .env (Essentiel !)

Le fichier .env est le cœur de votre configuration email. Il indique à Laravel comment envoyer vos emails.

Configuration Gmail

# Configuration SMTP pour Gmail MAIL_MAILER=smtp MAIL_HOST=smtp.gmail.com MAIL_PORT=587 MAIL_USERNAME=tonemail@gmail.com MAIL_PASSWORD=ton_mot_de_passe_app MAIL_ENCRYPTION=tls MAIL_FROM_ADDRESS=tonemail@gmail.com MAIL_FROM_NAME="MonApp"

⚠️ Sécurité Gmail

  1. Activez la validation en 2 étapes sur votre compte Google
  2. Générez un "mot de passe d'application" spécifique
  3. Utilisez ce mot de passe dans MAIL_PASSWORD
  4. Ne partagez jamais ces identifiants

Comprendre les Paramètres

MAIL_HOST : Adresse du serveur email (comme l'adresse de la poste)

MAIL_PORT : Port de communication (587 pour Gmail)

MAIL_USERNAME/PASSWORD : Vos identifiants d'authentification

MAIL_FROM_ADDRESS : Email expéditeur affiché

Email de Bienvenue

1. Créer une Notification

Une notification Laravel est un modèle d'email réutilisable. Créez-la avec cette commande :

php artisan make:notification WelcomeNotification

Structure de la Notification

public function via($notifiable) { return ['mail']; // Envoi par email } public function toMail($notifiable) { return (new MailMessage) ->subject('Bienvenue !') ->greeting('Salut ' . $notifiable->name . ' 👋') ->line('Merci de t\'être inscrit.') ->action('Visiter le site', url('/')) ->line('À bientôt !'); }

2. Envoyer la Notification

Dans votre contrôleur d'inscription, après la création de l'utilisateur :

// Après création de l'utilisateur $user->notify(new WelcomeNotification());

💡 Bonnes Pratiques

Envoyez l'email de bienvenue immédiatement après l'inscription pour créer un lien avec vos utilisateurs. Personnalisez le message avec leur nom pour une expérience plus chaleureuse.

Vérification d'Email

Pourquoi Vérifier les Emails ?

La vérification d'email garantit que l'adresse appartient réellement à l'utilisateur. Cela améliore la sécurité et la délivrabilité de vos emails.

Configuration

  1. Modifier le modèle User

    use Illuminate\Contracts\Auth\MustVerifyEmail; class User extends Authenticatable implements MustVerifyEmail

  2. Ajouter les routes de vérification

    // Dans routes/web.php Auth::routes(['verify' => true]);

  3. Protéger vos routes importantes

    Route::get('/dashboard', function () { return view('dashboard'); })->middleware(['auth', 'verified']);

⚠️ Middleware Important

auth : Utilisateur connecté

verified : Email vérifié

Ces deux middleware ensemble créent un système sécurisé.

Messages entre Utilisateurs

1. Notification de Message

php artisan make:notification NewMessageNotification

Implémentation de la Notification

class NewMessageNotification extends Notification { private $messageData; public function __construct($messageData) { $this->messageData = $messageData; } public function toMail($notifiable) { return (new MailMessage) ->subject('Nouveau message de ' . $this->messageData['sender_name']) ->greeting('Bonjour ' . $notifiable->name) ->line('Vous avez reçu un nouveau message :') ->line('"' . $this->messageData['message'] . '"') ->action('Voir le message', url('/messages')); } }

2. Contrôleur de Messages

public function send(Request $request) { // Validation $request->validate([ 'recipient_id' => 'required|exists:users,id', 'message' => 'required|string|max:500', ]); // Trouver le destinataire $recipient = User::findOrFail($request->recipient_id); // Préparer les données $messageData = [ 'sender_name' => auth()->user()->name, 'message' => $request->message, ]; // Envoyer la notification $recipient->notify(new NewMessageNotification($messageData)); return back()->with('success', 'Message envoyé !'); }

2. Queues pour les Performances

🚀 Pourquoi les Queues ?

Les queues permettent d'envoyer les emails en arrière-plan, évitant les ralentissements de votre application.

  1. Configuration

    # Dans .env QUEUE_CONNECTION=database

  2. Créer les tables

    php artisan queue:table php artisan migrate

  3. Rendre les notifications asynchrones

    class WelcomeNotification extends Notification implements ShouldQueue
